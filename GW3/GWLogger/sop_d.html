Themen:

Ein Gateway antwortet nicht

Gehen sie folgendermasse vor:
  1. Das Gateway mit Problemen identifizieren
  2. Die Art des Problems erkennen
  3. Neustart des Gateways mit CAESAR
  4. Ein Incident melden

Das Gateway mit Problemen identifizieren

Der einfachste Weg Probleme mit Gatways zu identifizieren ist mit CAESAR map (obere Menüleiste ⇒ Map). Die Map sollte so aussehen 1 wenn alles in Ordnung ist.
Eine Störung eines Gateways wird mit einem roten Hintergrund angezeigt. 2. Durch die Position des CAESAER (dieses Monitoring Tool) im Netzwerk, wird bei einem Problem bei einem Maschinen Gateway nicht nur dieses Gateway in rot angezeigt, sondern auch alle Gateways hinter dem defekten Gateway 3.
In diesem Fall liegt der Fehler am Gateway, das dem CAESAR am nächsten ist.

Mit einem Klick auf ein Gateay auf der Map wird zur Detailansicht umgeschlatet.

Die Art des Fehlers erkennen

Die Gateway Details lassen sich anzeigen, indem man:
  • Auf der Status Seite ein Gateway anklickt.
  • Auf der Map ein Gateway anklickt.
  • Das fehlerhafte Gateway auf dem "Errors" Feld anklickt.
Auf der Detailseite gibt es vier Grafiken, die einen Überblick über die letzten Minuten des Gateways geben, und die aktuellen Werte auf der rechten Seite anzeigen.

Bei einem Gateway, das nicht mehr antwortet, zeigen die Kurven eine Nulllinie und die aktuellen Werten zeigen eine 0 oder NULL. 5 Either wait for an automatic restart or restart manually.
Zeigt die Grafik ein ständiges auf und ab 6, dann ist dies meist ein Anzeichen dafür, dass ein Client das Gateway stört: erstellen sie in so einem Fall bitte ein "Service Now" Ticket mit so vielen Datails wie möglich. Bei hoher CPU Belastung des Gateways, wird ein Watchdog, der auf dem Gateway läuft, von Zeit zu Zeit einen Restart verursachen. 7. Im Falle einer CPU Last, die sich zyklisch wiederholt, ist das wiederum ein Zeichen für eine zu hohe Belastung durch einen Client. Wie zuvor sollten sie ein "Service Now" Ticket mit so vielen Details wie möglich erstellen.

Lösen sie in so einem Fall einen Restart mit CAESAR aus.

Von der entsprechenden Detail Seite kann ein Restart mit einem Klick auf den Button "Restart Gateway" ausgelöst werden. Warten sie danach einige Sekunden, dann sollte die "RunningTime" auf 0 gesetzt werden und wieder zu zählen beginnen.

Ein Incident melden:

Benutzen sie "Service Now" (css.psi.ch) um einen Incident zu melden. Folgende Informationen sind erforderlich:
  1. Im Titel: EPICS Gateway <Name des Gateways>
  2. Gateway Name
  3. Datum und Zeit des Vorfalls
  4. Wer hat das Problem bemerkt
  5. Was scheint nicht zu funktionieren
  6. Was wurde bereits unternommen
